                                         END-TO-END CONTROL FLOW
                                         -----------------------

 [1] LoginPage submit
      -> POST /api/auth/login
      -> session + store list returned
      -> StoreSelectModal choose store
      -> navigate /dashboard/:store/desktop

 [2] Dashboard boot
      -> GET /api/stores/:storeId/dashboard
      -> GET /api/stores/:storeId/workspaces/:workspaceId
      -> render rows + nav + operator list + workspace counts

 [3] Top-tabs leaf click
      -> workspaceMenuIntents resolves workflow intent
      -> workflow panel prefills fields
      -> submit workflow
      -> POST /api/stores/:storeId/workflow-actions
      -> API may create/update entity and/or queue StoreTask and StoreActivity
      -> UI refreshes activity/task rails

 [4] Service deep lane operations
      -> GET  /api/stores/:storeId/service-orders/:serviceOrderId
      -> POST /api/stores/:storeId/service-orders/:id/actions
      -> serviceOrderDetail mutation engine patches row + snapshot + activity semantics
      -> optional RO reference rewrite + notification synthesis

 [5] SLA governance
      -> POST /api/stores/:storeId/task-sla-policies
      -> POST /api/stores/:storeId/task-sla-policies/actions
      -> per-action SLA override, copy/preview/reset, retime open tasks, audit entries
